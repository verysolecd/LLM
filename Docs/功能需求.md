
导言：
你是自身架构师，当前仓库旨在实现一个本地大模型管理工具，使用网页界面A管理推理模型，语音模型与模型下载

本机配置：AMD7840HS CPU 集成780M显卡，windows11系统，
llama.cpp和whisper.cpp是模型拉起服务的工具, 分别用于启动llama.cpp模型和whisper.cpp模型,
_Models文件夹是推理模型文件和whisper使用的ggml文件存储的目录
文字转语音模型文件存储在TTS文件夹，使用index TTS-2和Qwen3-TTS的模型和开源项目来实现，但是管理页面要集成在网页界面A中
所有的目录路径，建议存在src\config.json中，以便其他代码模块读取
如果使用python，本机已经安装python3.11 和 uv,pip 使用清华源
其中推理模型页面.png是推理模型页面样式，推理模型需要可以启动和停止，并能显示当前运行的模型，并且拉起的模型服务的聊天界面，需要集成在其对应页面
语音模型页面.png是语音模型页面样式，
模型下载页面.png是模型下载页面样式，即顶部是模型仓库ID和模型文件ID的输入框，下面是嵌入的的modescope的模型搜索和下载页面，点击下载后需要能显示下载文件名字和进度
UI代码和服务代码请在src分文件夹存放，各个页面的代码和代码文件最好解耦，各服务的代码也要解耦，以便后续拓展和修改
