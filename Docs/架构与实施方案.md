# 本地大模型管理平台：架构与实施方案 (v2.0)

本项目是一个基于 **FastAPI + Vue 3 (Vite)** 构建的高性能本地大模型管理平台，旨在提供大语言模型 (LLM)、语音转文字 (Whisper)、语音合成 (TTS) 以及模型下载的一站式管理解决方案。

---

## 1. 核心架构设计

项目采用前后端分离架构，后端负责模型生命周期与重量级计算，前端提供沉浸式交互体验。

### 1.1 后端 (FastAPI)
- **API 驱动**：运行在端口 `5001`，通过 `api/v1` 提供路由分发。
- **生命周期管理**：在 `app.py` 中使用 `lifespan` 自动管理静态服务进程，并在启动时自动开启浏览器。
- **进程鲁棒性**：通过 `process_utils.py` 实现跨平台的进程树清理，确保模型退出时不留残留。
- **韧性 TTS 引擎**：`tts_engine.py` 具备自适应逻辑，自动识别并适配 Qwen-TTS 的 Base/Instruct 不同变体，确保服务稳定性。

### 1.2 前端 (Vue 3 + Vite)
- **工程化开发**：基于 **Vite + TypeScript + TailwindCSS v4**，支持热更新（HMR）。
- **集中式 API**：所有后端交互通过 `src/web-ui/src/api/` 统一管理，支持类型安全。
- **UI 特效**：采用 Glassmorphism（玻璃拟态）设计风格，提供响应式侧边栏、可拖拽日志面板及实时进度反馈。

## 3. 核心功能模块状态

### 3.1 LLM 推理 (Llama)
- **功能**：支持全模型列表管理、参数配置（ctx_size, seed 等）及实时对话。
- **交互**：集成流式渲染，支持多轮对话记录。

### 3.2 语音中心 (Audio & TTS)
- **TTS (语音合成)**：
  - 加载 Qwen-TTS 或其他兼容模型。
  - **自适应引擎**：`tts_engine.py` 会自动检测加载的模型类型（`base`, `custom_voice`, `voice_design`）。
  - **韧性与兼容性**：修复了 Base 模型不支持自定义音色时的 500 崩溃问题；现在系统会智能回退或向用户提供清晰的引导（如建议切换至 Instruct 版本）。
  - **新特性**：支持 9 种顶级发音人选择（限 Instruct 版），支持基于文本的“情绪指令”。
- **Whisper (翻译/识别)**：
  - 提供 Server, Stream, Command 三种运行模式切换。
  - 支持 `.bin` 格式模型的自动搜索。

### 3.3 下载监控 (Download Center)
- **模式**：基于 ModelScope SDK 实现。
- **进度反馈**：前端通过 `3s` 间隔轮询状态，实时展示下载百分比及带有微动效的进度条。
- **联动**：模型下载成功后，可手动或自动触发后端缓存刷新。

## 4. 运行引导

### 开发模式 (Dev)
1. **启动后端 API**: `python src/app.py` (运行在 5001)
2. **启动前端开发服务器**: 
   ```bash
   cd src/web-ui
   npm run dev
   ```
   访问 `http://localhost:5173`

### 生产模式 (Production)
1. **编译前端**: `npm run build`
2. **启动主程序**: `python src/app.py`
   程序将自动在 `8000` 端口启动 `http.server` 托管 `dist` 目录，并自动打开浏览器。
