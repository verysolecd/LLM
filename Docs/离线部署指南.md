# Python 离线环境极简部署指南

基于 `requirements.txt` 统一清单，下面这两条核心命令即可完成全套离线的打包与部署：

> 💡 **提示**：包存放文件夹（下文以 `Offline_Packages` 为例）可以任意命名，只需在两条命令中保持名字一致即可。

---

## 1. 打包阶段（于可以联网的机器的终端运行）

执行下面这条命令，它会根据清单，自动把所有的安装包（包含了依赖的依赖）统一下载到您指定的文件夹里。

```bash
pip download -d Offline_Packages -r requirements.txt --no-cache-dir
```
或者
```bash
Get-Content requirements.txt | ForEach-Object {
    if (-not [string]::IsNullOrWhiteSpace($_) -and not $_.StartsWith("#")) {
        pip download -d "Offline_Packages" $_ --platform win_amd64 --python-version 311 --only-binary=:all: --no-binary=jieba --no-binary=argbind --no-cache-dir
        if ($LASTEXITCODE -ne 0) {
            Write-Warning "⚠️ 下载失败：$_"
            break
        }
    }
}
```

---

## 2. 部署阶段（于离线/远程机器的新虚拟环境中运行）

把包含了 `requirements.txt` 以及存满安装包的夹子一起带到离线物理机上。在激活对应的 Python 虚拟环境后执行：

```bash
pip install --no-index --find-links ./Offline_Packages -r requirements.txt
```

*参数解析：*
* `--no-index`：强制无视网络环境，绝不向网络发起检索。
* `--find-links`：告诉 pip 必须且只能从我们指定的离线文件夹获取安装资源。

---

## ⚠️ 进阶：如何应对“网络机”与“离线机”环境不一致？
如果下载机与目标机的 **Python 版本不同**（例如 3.11 vs 3.10）或者 **操作系统不同**（比如用 Mac 为 Windows 机器下载包），直接运行上文默认的 `pip download` **绝对会报错安装失败**。
因为默认下载的底层 C++ 编译库 `.whl` 是和当前操作机绑定的（例如文件名包含 `cp311` 或 `win_amd64` 等字眼）。

要给不同的环境代下载，需要加上指定的约束参数（示例：目标机是 `Windows 64位` 系统且安装了 `Python 3.10`）：
```bash
pip download -d Offline_Packages -r requirements.txt --platform win_amd64 --python-version 310 --only-binary=:all:
```
*(注：当跨越系统/版本下载时，必须加上 `--only-binary=:all:` 以防止 pip 在您当下系统中常识构建源码包导致拉取不到依赖)*
